## 1. CLASS DEFINITIONS

### Layer
Represents a structural level of reality or system architecture.

### Principle
Represents a governing rule or meta-law of the system.

### Method
Represents a process used to evaluate, filter, or validate.

### Process
Represents a transformation or functional operation.

### Constraint
Represents a limiting condition required for system stability.

### StabilityMetric
Represents measurable system properties related to stability.

### FailureCondition
Represents conditions under which the system becomes unstable or collapses.

### CivilizationOutput
Represents the final system-level output.
