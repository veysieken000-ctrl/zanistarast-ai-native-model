# Contributing to Zanistarast AI Native Model

Thank you for your interest in contributing.

This repository is intended as an open, testable, verification-oriented framework. Contributions are welcome when they improve clarity, reproducibility, or testability.

---

# Contribution Principles

Every contribution should aim to improve at least one of the following:

- scientific clarity
- reproducibility
- verification quality
- traceability
- documentation
- implementation
- testing

---

# Pull Request Requirements

Before submitting a Pull Request:

- explain the purpose of the change
- describe expected effects
- provide supporting evidence where applicable
- document possible limitations

---

# Verification Requirements

Changes affecting the core framework should be evaluated against the repository's verification process.

Recommended validation sequence:

1. Deductive consistency
2. Inductive consistency
3. Rasterast Verification Filter
4. Ethical verification
5. Internal consistency
6. Traceability

---

# Code Style

Preferred characteristics:

- readable
- deterministic where possible
- documented
- modular
- testable

---

# Documentation

New concepts should include:

- definition
- motivation
- assumptions
- limitations
- examples

---

# Respect Existing Structure

Do not modify constitutional files without documenting the reason.

Core files should remain stable whenever possible.

---

# Testing

Whenever practical, include:

- unit tests
- validation examples
- reproducible demonstrations

---

# Community

Constructive criticism, verification attempts, replication studies, and alternative implementations are encouraged.

The goal is continuous improvement through transparent evaluation.
